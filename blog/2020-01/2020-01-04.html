    • git常用的指令

    设置用户：

    $ git config --global user.name "Your Name"
$ git config --global user.email "email@example.com

    # 目录操作

    • 创建一个空目录：bash mkdir <name>
    • 进入目录：bash cd <name>
    • 修改文件名：bash mv <oldName> <newName>
    • 显示当前目录：bash pwd
    • 显示当前目录下的文件名：bash ls
    • 初始化目录（这个目录变成Git可以管理的仓库）：bash git init

    # 基本提交

    • 将文件添加到暂存库：bash git add <file>
    • 将暂存库的文件提交到仓库：bash git commit -m '历史记录名'
    • 修改最近一次提交的历史记录：bash git commit --amend
    • 工作区此时的状态：bash git status
    • 查看file具体修改的内容：bash git diff <file>
    • 当提交时进入VIM界面时，操作命令为：i进入编辑模式；ESC退出编辑模式；wq保存并离开。

    # 查看某一分支的历史记录

    • 查看历史记录：bash git log
    • 查看简洁版历史记录：bash git log --pretty=oneline
    • 每条log只显示一行,显示number条：bash git log --oneline --number
    • 可以图形化地表示出分支合并历史：bash git log --oneline --graph
    • 可以显示特定分支的log：bash git log branchname
    • 可以查看在分支1,却不在分支2中的提交.^表示排除这个分支(Window下可能要给^branch2加上引号).：bash git log --oneline branch1 ^branch2
    • 会显示出tag信息：bash git log --decorate
    • 可以指定作者的提交历史：bash git log --author=[author name]
    • 根据提交时间筛选log：bash git log --since --before --until --after
    • 可以将merge的commits排除在外：bash git log --no-merges
    • 根据commit信息过滤log：bash git log --grep  （git log --grep=keywords）
    • 查看最近n次更新(-p展开每次提交的内容差异)：bash git log -p -<n>
    • 列出所有最近n周内的提交：bash git log --since=<n>.weeks
    • 使用图形化工具查阅提交历史：bash gitk

    # 版本操作

    • 退回到上一个版本：bash git reset --hard HEAD^
    • 退回到commit_id版本：bash git reset --hard commit_id(可以只写前几句)
    • 查看文件内容：bash cat <file>
    • 查看命令历史：bash git reflog
    • 查看工作区和版本库里面最新版本的区别：bash git diff HEAD -- <file>
    • 工作区的修改全部撤销：bash git checkout -- <file>
    • 缓存区的修改全部撤销：bash git reset HEAD <file>
    • 删除文件：bash git rm <file>(删除以后需要提交此操作)

    # 与远程联系

    • 列出每个远程库的简短名字：bash git remote
    • 显示对应的克隆地址：bash git remote -v
    • 如有多个远程仓库将其全部列出：bash cd grit
    • 创建SSH Key：bash ssh-keygen -t rsa -C "youremail@example.com"
    • 把一个已有的本地仓库与GitHub仓库连接：bash git remote add origin git@github.com:用户名/库名.git
    • 第一次推送master分支的所有内容：bash git push -u origin master（第一次需要-u，以后就可以把-u去掉）
    • 推送最新修改：bash git push (远程仓库名) (分支名)
    • 将一个远程GitHub仓库克隆到本地：bash git clone git@github.com:用户名/库名.git
    • 添加一个新的远程仓库，可以指定一个简单的名字，以便将来引用：bash git remote add [shortName] git@[url]/库名.git
    • 将一个远程仓库克隆到本地：bash git clone git@网址:文件位置/库名.git
    • 从远程仓库抓取数据到本地：bash git fetch [remote-name]
    • 修改某个远程仓库在本地的简称：bash git remote rename <oldName> <newName>
    • 移除本地仓库与对应的远端仓库的连接：bash git remote rm <本地name>
    • 命令抓取数据合并到本地：bash git pull
    • 查看某个远程仓库的详细信息：bash git remote show [remote-name]

    # 分支

    • 查看分支：bash git branch
    • 查看各个分支最后一个提交对象的信息：bash git branch -v
    • 查看已经与当前分支合并的分支：bash git branch --merged
    • 查看尚未与当前分支合并的分支：bash git branch --no-merged
    • 创建分支：bash git branch <name>
    • 切换分支：bash git checkout <name>
    • 创建+切换分支：bash git checkout -b <name>
    • 合并某分支到当前分支：bash git merge <name>(fast forward)（分支记录里不会有记录，历史记录里也不会有）
    • 合并某分支到当前分支：bash git merge --no-ff -m '历史记录' <name>(分支记录里有，历史记录里会有何时合并的)
    • 删除分支：bash git branch -d <name>
    • 强制性删除分支：bash git branch -D <name>
    • 查看分支合并图：bash git log --graph

    # git栈缓存

    • 备份当前的工作区的内容，从最近的一次提交中读取相关内容，让工作区保证和上次提交的内容一致。同时，将当前的工作区内容保存到Git栈中： bash git stash
    • 显示Git栈内的所有备份： bash git stash list
    • 删除储藏区已回复内容：bash git stash drop
    • 从Git栈中读取最近一次保存的内容，恢复工作区的相关内容同时删掉Git栈中 部分：bash git stash pop
    • 清空Git栈：bash git stash clear

    # 标签

    • 为HEAD新建一个标签，即为最新的提交记录：bash git tag <name>
    • 为commit id新建一个标签： bash git tag <name> commit id
    • 可以指定标签信息：bash git tag -a <tagname> -m "blablabla..."
    • 可以用PGP签名标签：bash git tag -s <tagname> -m "blablabla..."
    • 查看所有标签：bash git tag
    • 查看相应标签的版本信息：bash git show <name>
    • 向远程推送一个本地标签：bash git push origin <tagname>
    • 向远程推送全部未推送过的本地标签：bash git push origin --tags
    • 可以删除一个本地标签：bash git tag -d <tagname>
    • 可以删除一个远程标签：bash git push origin :refs/tags/<tagname>
